## Descripció del projecte

Aquest projecte consisteix en un sistema de gestió d'usuaris desenvolupat amb HTML, CSS (usant Skeleton) i Javascript. Permet el registre, login i control d'accés dels usuaris mitjançant un arxiu `.json` per emmagatzemar la informació.

## Resum de conceptes Scrum

### Rols
- **Product Owner:** Responsable de definir les funcionalitats del producte i prioritzar les tasques segons el valor per al client.
- **Scrum Master:** Facilita el procés Scrum, elimina impediments i ajuda l'equip a seguir les pràctiques àgils.
- **Equip de desenvolupament:** Format per professionals multidisciplinaris que creen el producte i s'autoorganitzen per assolir els objectius del Sprint.

### Esdeveniments
- **Sprint Planning:** Reunió per planificar el treball del Sprint i definir l'objectiu.
- **Daily Scrum:** Reunió diària breu per sincronitzar l'equip i detectar obstacles.
- **Sprint Review:** Revisió del treball fet durant el Sprint amb el Product Owner i altres interessats.
- **Retrospective:** Reunió per reflexionar sobre el procés i identificar millores per al proper Sprint.

### Documents
- **Product Backlog:** Llista prioritzada de totes les funcionalitats, millores i correccions pendents del producte.
- **Sprint Backlog:** Subconjunt del Product Backlog seleccionat per al Sprint, amb les tasques a realitzar.
- **Burn-down chart:** Gràfic que mostra la quantitat de treball pendent al llarg del Sprint, ajudant a visualitzar el progrés.
